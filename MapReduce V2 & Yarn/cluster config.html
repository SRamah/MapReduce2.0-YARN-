Master BDSAS 2017

Hadoop 2.7.3 multi-node cluster setup on Ubuntu 14.04 LTS

Context


Multi-Node Hadoop 2.7.3 cluster (all Virtual Machines); 1 Master + 2 Slaves; Data nodes configured only on Slaves;

Steps to be followed


Consider the assignment of names and IP to the cluster as follows:

Node
Hostname
IP Address
Namenode -> Master
HMaster
10.211.55.5
Datanode -> Slaves
HSlave0(HMaster)
localhost (HMaster)
HSlave1
10.211.55.9

1. Clone Hadoop Single node VM as HMaster

2. To be done in the HMaster [For creating slaves]

a) Open terminal; add the node details in hosts file
$sudo gedit /etc/hosts
HMaster 10.211.55.5
HSlave1 10.211.55.9

b) Edit the hostname
$sudo gedit /etc/hostname
HMaster

c) To update hadoop configuration files, go to /usr/local/hadoop/etc/hadoop folder
$cd /usr/local/hadoop/etc/hadoop

d) Edit core-site.xml file
$sudo gedit core-site.xml
Replace localhost as HMaster

e) Edit hdfs-site.xml file
$sudo gedit hdfs-site.xml
Replace value 1 as 2 (represents the number of datanodes)
Remove dfs.namenode.name.dir property section

f) Edit yarn-site.xml file
$sudo gedit yarn-site.xml
To be added within <configuration> tag
<property>
 <name>yarn.resourcemanager.resource-tracker.address</name>
 <value>HMaster:8025</value>
</property>
<property>
 <name>yarn.resourcemanager.scheduler.address</name>
 <value>HMaster:8030</value>
</property>
<property>
 <name>yarn.resourcemanager.address</name>
 <value>HMaster:8050</value>
</property>

g) Create datanodes and give ownership
$cd
$sudo rm -rf /usr/local/hadoop/hadoop_data
$sudo mkdir -p /usr/local/hadoop/hadoop_data/hdfs/datanode
$sudo chown -R sawmya:sawmya /usr/local/hadoop

h) Reboot HMaster

i) Clone HMaster as HSlave1


3. To be done in the slave [HSlave1]

a) Open terminal; Edit the hostname
$sudo gedit /etc/hostname
HSlave<number as approriate>

b) Reboot all slaves


4. To be done in the HMaster [changing the configurations done to clone the slave]

a) Open terminal; add master and slave details
$sudo gedit /usr/local/hadoop/etc/hadoop/masters
HMaster
$sudo gedit /usr/local/hadoop/etc/hadoop/slaves
Remove localhost and add
HSlave1

b) Edit the hdfs-site.xml
$sudo gedit /usr/local/hadoop/etc/hadoop/hdfs-site.xml
Add the property dfs.namenode.name.dir

c) Create namenode and give ownership
$sudo mkdir -p /usr/local/hadoop/hadoop_data/hdfs/namenode
$sudo chown -R USER:GROUP /usr/local/hadoop

d) Format the namenode
$hadoop namenode –format
The exit status should be 0.

e) Start the hadoop daemons
$start-dfs.sh
$start-yarn.sh

f) To see the daemons that are running
$jps
In Master, it should display SecondaryNamenode, Namenode, ResourceManager, Datanode, NodeManager and jps
In Slaves, it should display Datanode, NodeManager and jps

g) To view the hadoop details in browser
http://Master:50070/
http://Master:8088/
